# BB84 partial protocol claim

This benchmark documents a **partial** BB84 sifted-key protocol claim in `artifacts/protocol.json`. The machine spec uses `mode: negative` to state that full QKD security and privacy are **not** claimed checked.

Human review confirms honest partial labeling: no sifted-key rate proof, no eavesdropper model formalization, and no privacy-amplification claim. The protocol artifact describes rounds and sifting at a schematic level only.

Reference promotion is blocked by design until independent security evidence exists.
